# RULING-OBJECT-SYSTEM

Register: standing ruling, decided with Travis 2026-07-19 during the proactivity reference iteration. Governs webapp, mobile, and desktop. Supersedes the per-surface guesswork that produced three correction passes on one surface.

## The ruling

1. Three systems, three scopes, no bespoke surfaces:
   - The shell is Int UI (toolbar, stripes, tabs, editor well, status bar). Unchanged.
   - Tabular lenses are the Twenty system (record tables, dense rows, side-panel inspector). Already in force.
   - Object surfaces are the Capacities system: typed icon tiles with tint pairs, warm content ground (Clay light on light), lifted white cards with soft shadows over the ground, plain bold section titles (no panel chrome strips on content planes), generous-but-dense rows, one view bar.
   A surface that is not a shell region, a table, or an object surface does not exist; it decomposes into these.
2. Kind tiles carry icons, never letters, never author rails. Tint pairs per kind, spread across the wheel so adjacent kinds never clash: stakes teal, watches indigo, judgments violet (reserved), responses slate (reserved). Gold is the learned and grant register and appears only in badges, never as a kind tint. The clash observed (gold tiles beside teal and green) is the defect this rule prevents.
3. Authorship renders as face plus a small author chip on human-authored items (oxblood tint chip reading "yours"), and as the tile tint on human-authored rows. No rails.
4. Depth on light content planes is lift: soft diffuse card shadows (approximately 0 1px 2px at 5 percent plus 0 6px 18px at 7 percent) over hairline borders, on a textured ground (paper grain at roughly 4 to 5 percent). The Int UI depth-by-value law continues to govern the shell; content planes lift. Both themes keep the contrast gate.
5. Control rows consolidate: one view bar per surface (breadcrumb, view segmented control, source or type dropdown, filters) in a single 44px row. View switchers never occupy their own row and never push content.
6. Sources and type scoping are dropdown selectors in the view bar, not panels of boxes.
7. Plain-language compile affordances dock at the bottom as a floating bounded instrument (measured width, labeled, its own visual identity), never inline in rows and never resembling the Chat Composer.
8. Progress renders as meters and sparklines, not sentences; state phrases render as badges.

## Reference renderings are the spec

The iteration artifact (proactivity-reference-v2.html, committed beside this ruling when ratified) is the acceptance target: implementing sessions match the rendering, and the review is a side-by-side capture. Prose describes intent; renderings decide disputes. Any surface correction that fails twice in prose moves to a rendering before a third attempt.
